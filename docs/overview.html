Runtime pipeline

01

Five sequential stages drive the runtime. Each stage is a single Python command; there is no daemon, no scheduler, and no build step. The pipeline is designed so every stage can be re-run independently.

pipeline.svg · five stages · no external dependencies

Quick start

02

Install the package in editable mode (pip install -e ".[dev]"), then run the six commands below in order on first setup:

# 1. Validate the seed question bank
python -m cca_f_study.validate_questions 02-question-bank/seed/sample-questions.jsonl

# 2. Submit a mock-exam attempt
python 04-exam-runner/submit_attempt.py \
  --questions 02-question-bank/seed/sample-questions.jsonl \
  --answers   examples/attempts/sample-answers.json \
  --out       05-learning-data/attempts/sample-attempt.json

# 3. Score the attempt
python 04-exam-runner/score_attempt.py 05-learning-data/attempts/sample-attempt.json

# 4. Export dashboard data
python 04-exam-runner/export_dashboard_data.py \
  --attempts   05-learning-data/attempts \
  --lab-status 05-learning-data/lab-status.json \
  --out        06-dashboard/data/dashboard-data.json \
  --now        2026-05-12T00:30:00Z

# 5. Serve the static dashboard
python -m http.server 8000 -d 06-dashboard
# then open http://localhost:8000/static/

# 6. Run the full test suite
pytest

Safety and boundaries

06

The MVP boundary is intentionally tight. These rules are enforced by regression tests and will not be relaxed without a new spec revision:

  • No PDF parsing at runtime — the source guide is read-only evidence
  • No HTTP requests at any stage — verified by test_no_network_calls.py
  • No API keys or secrets required
  • Generated questions carry status != "official" always
  • Dashboard is static HTML/CSS/JS — no build step, no CDN
  • Dashboard-data export is byte-identical across re-runs
  • Out of scope: adaptive question generation via Claude API
  • Out of scope: multi-user or remote deployment
  • Out of scope: CI/CD pipeline (all checks are local pytest)
  • Out of scope: persistent database (all data is JSON/JSONL)
