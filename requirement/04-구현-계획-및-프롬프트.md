가능합니다. 다만 **처음부터 “PDF 기반 자동 문제 생성 + 모의고사 앱 + 대시보드 + 실습 코드 전체”를 한 번에 구현하려고 하면 너무 큽니다.**
우선은 en PDF 하나만 repo에 넣어두고, **학습 운영 MVP**를 만드는 게 맞습니다.

GitHub 기준으로 해당 CCA-F 저장소는 `guide_en.MD`와 `pdf/guide_en.pdf`를 제공하고, README에서도 Practical Exercises로 tool design, MCP integration, structured output, context management, reliability를 연습하라고 되어 있습니다. 그러므로 첫 구현은 “PDF를 런타임에서 분석하는 시스템”이 아니라, **PDF를 source artifact로 보관하고, 문제은행·풀이 제출·점수 계산·대시보드 데이터를 운영하는 시스템**으로 잡는 게 좋습니다.

## 결론

**Superpowers 쓰는 게 좋습니다.**
단, 처음부터 거창한 multi-agent 체계까지는 필요 없고, Claude Code에서 다음 순서로만 쓰면 충분합니다.

```text
1. Spec 작성
2. Implementation Plan 작성
3. MVP 구현
4. Validation
5. Dashboard Design
6. Dashboard 구현
```

한 개 프롬프트로는 하지 않는 게 좋습니다.
대신 **6개 프롬프트 체인**으로 끊어서 구현하는 게 안전합니다.

## 3. Superpowers를 쓴다면 이렇게

너의 경우에는 **Superpowers식 개발이 맞습니다.**
이유는 구현 대상이 단순 파일 몇 개가 아니라, “학습 데이터 운영 시스템”이기 때문입니다.

추천 흐름:


다만 “superpowers”를 별도 거대한 체계로 만들 필요는 없습니다.

## 12. 초기 CLAUDE.md에 꼭 넣을 내용

최소한 이 정도는 넣고 시작하세요.

```markdown
# CCA-F Study Runtime

This repository implements a local study runtime for Claude Certified Architect — Foundations.

## MVP boundary

The initial repository contains only one source PDF:

- `01-sources/en/guide_en.pdf`

The PDF is a read-only source artifact.  
Runtime scoring and dashboard generation must not depend on parsing the PDF.

## Runtime source of truth

Runtime uses:

- question bank JSONL
- submitted attempt JSON
- lab status JSON
- dashboard-data JSON

## Scoring

Default scoring:

- `scaled_score = round(100 + (correct / total) * 900)`
- `pass_mark = 720`

Dashboard must show:

- raw score
- scaled score
- pass/fail
- gap to pass mark
- domain breakdown
- scenario breakdown
- weak concept tags
- lab progress
- trend

## Domain weights

- D1 Agentic Architecture & Orchestration: 27%
- D2 Tool Design & MCP Integration: 18%
- D3 Claude Code Configuration & Workflows: 20%
- D4 Prompt Engineering & Structured Output: 20%
- D5 Context Management & Reliability: 15%

## Question rules

Each question must have:

- id
- source
- domain
- scenario
- difficulty
- stem
- choices A-D
- answer
- explanation
- concept_tags
- status

Generated or community questions are unofficial.

## Safety

- Do not call external APIs in MVP.
- Do not require API keys.
- Do not mutate source PDF.
- Do not claim any generated question is official.
- Keep dashboard static until MVP is complete.
```
