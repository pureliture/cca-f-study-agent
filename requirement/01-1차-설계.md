Skilljar 강의와 실습 노트북을 **"이론(PDF) ↔ 실습(Lab) ↔ 복습(노트)"**이 1:1로 연결되는 구조로 설계하는 것이 핵심입니다. NotebookLM에는 이론 PDF만 올리되, 실습 파일의 위치와 맥락을 인덱스로 연결해두면 학습 흐름이 끊기지 않습니다.

1. 설계 원칙

원칙

설명

번호 동기화

이론 PDF와 실습 폴더에 동일한 번호(01-, 02-)를 부여해 정렬 시 자동 매핑

단일 출처(Single Source of Truth)

PDF는 NotebookLM용으로만 읽기 전용 보관. 실습 코드는 Jupyter Lab/VS Code에서만 편집

맥락 인덱스

NotebookLM이 PDF 내용에 대해 질문했을 때, 어떤 실습 폴더로 가야 하는지 알려주는 메타데이터 파일을 함께 업로드

모듈 단위 격리

각 강의를 독립 폴더로 분리해, 강의가 업데이트되어도 특정 모듈만 교체 가능

3. 핵심 매커니즘: 이론과 실습을 연결하는 인덱스

NotebookLM에 PDF만 올리면, "이 개념의 실습은 어디 있지?"라고 물었을 때 답을 찾기 어렵습니다.**00-meta/notebooklm-source-index.md**를 PDF와 함께 업로드하면, NotebookLM이 파일 간 관계를 이해합니다.

# Anthropic Academy Source Index

## Module 01: Intro to Prompting
- Theory: `01-theory-notebooklm/01-intro-to-prompting.pdf`
- Lab Location: `02-labs/01-intro-to-prompting/`
- Key Concepts: zero-shot, few-shot, chain-of-thought
- Learning Goal: 기본 프롬프트 구조 실습

## Module 02: Running the Eval
- Theory: `01-theory-notebooklm/02-running-the-eval.pdf`
- Lab Location: `02-labs/02-running-the-eval/`
- Key Concepts: evaluation pipeline, test case, dataset
- Learning Goal: run_prompt / run_test_case / run_eval 파이프라인 구현
- Note: 채점기는 아직 placeholder(score=10) 상태. 실제 채점은 Module 03에서 연결.

## Module 03: Model Based Grading
- Theory: `01-theory-notebooklm/03-model-based-grading.pdf`
- Lab Location: `02-labs/03-model-based-grading/`
- Key Concepts: model-as-a-judge, JSON prefill, stop_sequences
- Learning Goal: grade_by_model 함수 구현 및 Module 02 파이프라인과 통합

팁: 이 파일을 NotebookLM에 업로드하면, "Module 2의 실습은 어디서 하나요?" 또는 "Grader를 구현하려면 이전에 어떤 파이프라인이 필요했나요?" 같은 질문에 PDF 내용 + 이 인덱스를 함께 참조합니다.

7. 주의할 점

PDF 편집 금지: 01-theory-notebooklm/의 PDF는 NotebookLM 재업로드를 위해 원본 그대로 보관. 개인 메모는 03-my-notes/에 별도 작성

노트북 출력값 관리: Git 등으로 버전 관리 시, nbstripout이나 Jupyter의 "Clear All Outputs" 후 저장하면 diff가 깔끔해집니다

API 키 노출 방지: _shared/의 설정 파일은 .env 또는 .gitignore 처리된 외부 파일에서 로드하고, 노트북에는 하드코딩하지 마세요
